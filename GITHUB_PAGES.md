# Инструкция по размещению на GitHub Pages

## ✅ Да, ваш сайт будет работать на GitHub Pages!

Все файлы настроены правильно для работы на GitHub Pages. Вот что нужно сделать:

## Быстрый старт

1. **Создайте репозиторий на GitHub**
   - Зайдите на https://github.com
   - Нажмите "New repository"
   - Назовите репозиторий (например: `monument-shibanov`)
   - Создайте репозиторий

2. **Загрузите файлы**
   
   Через командную строку:
   ```bash
   cd /home/konstantsin/apps/otez-project
   git init
   git add .
   git commit -m "Памятник Г.И. Шибанову"
   git branch -M main
   git remote add origin https://github.com/ВАШ-USERNAME/НАЗВАНИЕ-РЕПОЗИТОРИЯ.git
   git push -u origin main
   ```
   
   Или через веб-интерфейс GitHub:
   - Нажмите "uploading an existing file"
   - Перетащите все файлы из папки проекта
   - Нажмите "Commit changes"

3. **Включите GitHub Pages**
   - Settings → Pages
   - Source: выберите `main` ветку
   - Folder: `/ (root)`
   - Save

4. **Готово!**
   - Через 1-2 минуты сайт будет доступен по адресу:
   - `https://ваш-username.github.io/название-репозитория/`

## Важно знать

- **Размер файла**: model.glb весит 26MB - это нормально, но загрузка может занять несколько секунд
- **HTTPS**: GitHub Pages автоматически использует HTTPS, поэтому CORS проблемы не будет
- **Обновления**: После каждого `git push` сайт обновится автоматически (1-2 минуты)

## Структура файлов для GitHub

```
репозиторий/
├── index.html          ← Главная страница
├── styles.css          ← Стили
├── model-viewer.js     ← JavaScript для 3D
├── model.glb           ← 3D модель (26MB)
├── .nojekyll           ← Отключает Jekyll (уже создан)
├── .gitignore          ← Игнорируемые файлы
└── README.md           ← Описание проекта
```

Все файлы должны быть в **корне репозитория** (не в подпапках)!

## Решение проблем

**Модель не загружается:**
- Проверьте, что файл `model.glb` загружен в репозиторий
- Откройте консоль браузера (F12) и проверьте ошибки
- Убедитесь, что используете HTTPS (не HTTP)

**Сайт не обновляется:**
- Подождите 1-2 минуты после push
- Очистите кэш браузера (Ctrl+F5)

**Стили не работают:**
- Проверьте, что файл `styles.css` в корне репозитория
- Проверьте путь в `<link rel="stylesheet" href="styles.css">`
